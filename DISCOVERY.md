This repository is a compact, cross-platform process-environment conformance harness: the POSIX side (`execve` + `envp`) and the Windows side (`CreateProcessW` + `lpEnvironment`) encode the same semantics checks for inheritance, replacement, append, omission, ordering, duplicate keys, malformed entries, and Unicode environment blocks, which makes it directly reusable as acceptance criteria for any `organvm` process runner or VM. Its highest latent value is as a low-cost reliability and security gate to prevent environment-contamination regressions in host/guest launches, where these APIs are common and easy to get subtly wrong.

Single best first task: publish these two C harnesses as a shared, versioned conformance test package with standardized golden expectations and wire it into every language/runtime CI matrix so environment-propagation behavior is validated before runner releases.
